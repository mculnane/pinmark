# CLAUDE.md

Guidance for Claude Code working in this repository.

## What this is

Pinmark — a Safari Web Extension (Manifest V3) that saves the current tab to
Pinboard with frequency-ranked tag autocomplete. Public repo:
<https://github.com/mculnane/pinmark>. Personal/local use; not on the App Store.

## Layout

- `src/` — the web extension, and the **source of truth**. Edit here.
  - `manifest.json` — MV3 manifest
  - `popup/` — popover UI; `popup.html` loads `popup.js` as an ES module
  - `background/` — service worker (classic script — see gotchas)
  - `lib/` — `browser.js` (cross-browser API shim), `storage.js`,
    `pinboard-api.js` (transport-swappable client), `tag-autocomplete.js`
    (pure ranking/tokenizing logic)
  - `icons/` — generated PNGs
- `xcode/` — Xcode container app + extension target, generated by
  `scripts/convert.sh`. It **references** `src/` via relative paths (no
  `--copy-resources`), so editing `src/` and rebuilding picks up the changes.
- `tests/` — `*.test.mjs`, runnable without Node (see below)
- `scripts/` — `convert.sh` (regenerate the Xcode project), `generate_icons.py`

## Build & run

1. `open xcode/Pinmark/Pinmark.xcodeproj` (run `./scripts/convert.sh` first if
   `xcode/` is missing or the manifest structure changed).
2. Set the signing Team on both targets (`Pinmark`, `Pinmark Extension`) with
   automatic signing.
3. ⌘R, then enable in Safari → Settings → Extensions (and Develop → Allow
   unsigned extensions when signed with a free Apple ID).

**Safari loads the extension from the built `.appex`, not from `src/` directly.**
Rebuild (⌘R) for any JS/HTML/CSS change to take effect.

## Tests

There is no Node on the primary machine; tests run under JavaScriptCore:

```sh
JSC=/System/Library/Frameworks/JavaScriptCore.framework/Versions/A/Helpers/jsc
"$JSC" -m tests/tag-autocomplete.test.mjs   # autocomplete ranking/tokenizing
"$JSC" -m tests/pinboard-api.test.mjs       # API error classification
```

`jsc` lacks `URL`/`URLSearchParams`/`fetch` — tests shim or inject these. To
syntax-check a module: `"$JSC" -m <file>` (a `SyntaxError` is a real problem; a
runtime `ReferenceError` for `document`/`browser` just means there's no DOM under
jsc, and the syntax is fine).

## Conventions & gotchas

- **The API layer is transport-swappable.** The UI only calls
  `createPinboardClient()` from `lib/pinboard-api.js`. To route through an MCP
  server later, add a transport with the same `request(path, params)` method —
  don't touch the UI.
- **`pinboard-api.js` is the only module that knows the transport.** Its errors
  carry a stable `code` (`unauthorized` / `rate_limited` / `server_error` /
  `network`); the popup switches on `code`, never on message text.
- **The background service worker must stay self-contained** — no ES `import`.
  Safari's support for module service workers is unreliable, so
  `background/background.js` duplicates the few storage keys it needs. Keep them
  in sync with `lib/storage.js`.
- **Rate limit:** Pinboard asks for ≥3s between calls; the client serialises and
  spaces requests. Don't remove the limiter.
- **Token** lives only in `browser.storage.local`, entered via the popup's
  settings panel. Never read it from disk, never commit it.
- **Signing / `project.pbxproj`:** the committed `DEVELOPMENT_TEAM` is empty on
  purpose (this is a public repo). A local build re-adds your own Team ID, but the
  file is marked `skip-worktree` so it won't be staged. Don't commit a real Team
  ID. To make a genuine project-structure change, first run
  `git update-index --no-skip-worktree xcode/Pinmark/Pinmark.xcodeproj/project.pbxproj`.
- **A free Apple ID signing certificate expires every 7 days** — rebuild to
  refresh.

## Icons

`python3 scripts/generate_icons.py` renders a 512px master (pure-Python PNG
encoder, no dependencies) and downscales the rest with `sips`.
